contentcheck:check

Full name:

net.sf.buildbox.maven:contentcheck-maven-plugin:1-SNAPSHOT:check

Description:

Checks the archive content according to an authoritative source. This authoritative source defines set of allowed files in the archive.

Attributes:

  • Requires a Maven project to be executed.
  • The goal is thread-safe and supports parallel builds.
  • Binds by default to the lifecycle phase: verify.

Optional Parameters

Name Type Since Description
archive File - The archive file to be checked. You should specify either archive or directory. Directory takes a precedence before archive.
Default value is: ${project.build.directory}/${project.build.finalName}.${project.packaging}.
checkFilesPattern String - An Ant like file pattern. If this roperty is present only files matching that pattern are checked. Otherwise all JAR files are checked.
Default value is: **\/*.jar.
contentListing File - The file with list of expected files. Each line in that file represents one pathname entry. Empty lines and comments (starting with '#') are ignored.
directory File - The directory to be checked. You should specify either archive or directory. Directory parameter has no default value therefore it takes a precedence before archive.
failOnMissing boolean - If true, stops the build when there is any file missing.
Default value is: false.
failOnUnexpected boolean - If true, stops the build when there is any unexpected file.
Default value is: true.
ignoreVendorArchives boolean - If true, doesn't check vendor JAR files. A vendor JAR file is determined by a value (vendorId) in its manifest key (manifestVendorEntry).
Default value is: false.
manifestVendorEntry String - The name of manifest entry that holds vendor's identification
Default value is: Implementation-Vendor-Id.
msgMissing String - Message used to report missing entry - uses the Formatter syntax to embed entry name.
Default value is: File is expected but not found: %s.
msgUnexpected String - Message used to report unexpected entry - uses the Formatter syntax to embed entry name.
Default value is: Found unexpected file: %s.
skipPOMPackaging boolean - If true, modules with POM packaging are skipped (excluded from the content check).
Default value is: false.
vendorId String - The vendor identification. This value is used for JAR's manifest checking when ignoreVendorArchives is turned on.
