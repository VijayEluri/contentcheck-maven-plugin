contentcheck:show-licenses

Full name:

net.sf.buildbox.maven:contentcheck-maven-plugin:1-SNAPSHOT:show-licenses

Description:

This MOJO shows license information for a source entries. Artifact resolving and the rest of Maven repo magic taken from Maven Project Info Reports Plugin. The MOJO shows license information only for entities that matches criteria defined by getCheckFilesPattern() and isIgnoreVendorArchives().

The MOJO's output is printed by default to console log and also to CSV file ${project.build.directory}/licenses.csv.

License information is gathered from dependency's POM, but a project may define additional mapping between files in a project archive and licenses. This mapping file is src/main/license.mapping.json and its structure is JSON.

Additional license information


{
"licenses": [
      {
         "name" : "License name",
          "url"  : "License text URL",
          "files": [
              "file name"
           ]
      }
 ]
}

Example


{
   "licenses": [
      {
          "name"  : "The LGPL license 2.1",
          "url"   : "http://www.gnu.org/licenses/lgpl-2.1.html",
          "files" : [
                        "aspectwerkz-nodeps-jdk5-2.2.1.jar"
          ]
      },
      {
          "name"  : "The public domain",
          "url"   : "http://creativecommons.org/licenses/publicdomain/",
          "files" : [
              "jsr166x-1.0.jar",
              "xyz.jar"
          ]
      }
   ]
}

Attributes:

  • Requires a Maven project to be executed.

Optional Parameters

Name Type Since Description
archive File - The archive file to be checked. You should specify either archive or directory. Directory takes a precedence before archive.
Default value is: ${project.build.directory}/${project.build.finalName}.${project.packaging}.
checkFilesPattern String - An Ant like file pattern. If this roperty is present only files matching that pattern are checked. Otherwise all JAR files are checked.
Default value is: **/*.jar.
contentListing File - The file with list of expected files. Each line in that file represents one pathname entry. Empty lines and comments (starting with '#') are ignored.
Default value is: src/main/content.txt.
csvOutput boolean - If true print the result of check to a CSV file in project build directory. See csvOutputFile
Default value is: true.
csvOutputFile File - The CSV output file that is used when csvOutput is turned on.
Default value is: ${project.build.directory}/licenses.csv.
defaultBundleForPOMPacking File - The archive file to be checked
Default value is: ${project.build.directory}/${project.build.finalName}.zip.
directory File - The directory to be checked. You should specify either archive or directory. Directory parameter has no default value therefore it takes a precedence before archive.
ignoreVendorArchives boolean - If true, doesn't check vendor JAR files. A vendor JAR file is determined by a value (vendorId) in its manifest key (manifestVendorEntry).
Default value is: false.
licenseMappingFile File - The license mapping file. This file may define additional license information for JARs that are not recognized.
Default value is: src/main/license.mapping.json.
User property is: licenseMappingFile.
manifestVendorEntry String - The name of manifest entry that holds vendor's identification
Default value is: Implementation-Vendor-Id.
vendorId String - The vendor identification. This value is used for JAR's manifest checking when ignoreVendorArchives is turned on.
